Error patterns that indicate a fatal, non-retryable failure.

These are checked against the error message string returned by the StackQL engine. If any pattern matches, the operation is aborted immediately rather than retried.

Two categories:

  1. Network errors - The request never reached the API. Any result from a query in this state is untrustworthy (e.g., an exists check returning empty could cause a duplicate resource to be created).

  2. HTTP status errors - The request reached the API but the response indicates an unrecoverable problem (auth failure, forbidden, etc.). 404 is explicitly excluded as it’s normal for exists checks.
